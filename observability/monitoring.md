# Observability and monitoring

This document defines the global observability expectations.

## Goals

The team should be able to understand whether a system is healthy and investigate problems without relying on guesswork.

## Minimum project documentation

Each project should document:

- where production logs are available
- where build and deployment logs are available
- how errors are reported
- what analytics or monitoring tools are used
- who receives alerts, if alerts exist
- known operational risks

## Operational ownership

If a project is live, someone should know:

- how to confirm it is working
- how to investigate an error
- how to rollback a bad deployment
- how to contact the right people when support is needed
